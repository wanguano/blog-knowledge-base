# Dev Knowledge Base (VitePress)

基于 VitePress 构建的高质量技术知识库，旨在记录开发过程中的**技术难点**、**踩坑经验**、**学习笔记**以及**可复用的 Hooks 和组件**。

## 🚀 项目初衷

- **沉淀知识**：将碎片化的学习笔记整理成系统化的文档。
- **避免重复**：详细记录“踩坑”过程和解决方案，提升团队/个人复用效率。
- **最佳实践**：维护一套高质量的 React/Vue Hooks 和通用组件库。

## 📂 目录结构

```text
docs/
├── .vitepress/          # VitePress 核心配置
├── posts/               # 核心文章
│   ├── 技术难点/        # 深度技术方案解析
│   ├── 踩坑记录/        # 错误现场与快速修复
│   └── 学习笔记/        # 读书笔记、技术总结
├── hooks/               # 可复用 Hooks
│   ├── react/
│   └── vue/
├── components/          # 通用组件库
├── utils/               # 工具函数
└── public/              # 静态资源（图片、下载文件）
```

## 🛠️ 快速开始

### 安装依赖
```bash
pnpm install
```

### 开发环境
```bash
pnpm run dev
```

### 生产构建
```bash
pnpm run build
```

## 📝 编写规范

请遵循以下文档模板（详见 `.cursorrules`）：

1.  **技术难点**：包含背景挑战、方案对比、核心代码、效果对比。
2.  **踩坑记录**：包含问题现象、环境信息、解决方案、原理分析、预防措施。
3.  **React Hook**：包含功能说明、API 文档、使用示例。
4.  **前端组件**：包含组件说明、代码演示、Props 详情。

## 🤝 贡献指南

1. 在对应的目录下创建 `.md` 文件。
2. 使用清晰的命名规范（例如：`react-performance-optimization.md`）。
3. 确保包含完整的 Frontmatter 描述信息。
