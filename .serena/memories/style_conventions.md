# Style & Conventions

- All blog posts, component docs, and hook docs must follow the specific Markdown templates defined in `.cursorrules`.
- Markdown files should be placed in their respective directories (`docs/posts/`, `docs/hooks/`, `docs/components/`).
- Use Frontmatter (YAML) at the top of markdown files for metadata (title, description, date, tags).
- Use `::: code-group` for comparing code options.
- Name files descriptively using kebab-case (e.g., `react-performance-optimization.md`). Do not use generic names like `doc1.md`.
- Include functional examples or snippets where appropriate.
